# Submission Playbook — President Tech Award & President AI Award 2026

Пошаговая подготовка к обнаруженной 5-стадийной воронке отбора (Заявка → Менторство → Demo Day
→ Fact-Checking → Pitch Polish → Grand Final). Контекст, факты и решения по позиционированию —
в [`AWARDS_STRATEGY_2026.md`](./AWARDS_STRATEGY_2026.md). Доказательная база для Fact-Checking —
в [`JUDGE_EVIDENCE_PACKET.md`](./JUDGE_EVIDENCE_PACKET.md).

**Две отдельные заявки, два нарратива:**

| | President Tech Award — Incubation | President AI Award — Образование |
|---|---|---|
| Продукт | FundingPro Business | Opportunities Lab |
| Сегмент | МСБ, предприниматели | Молодёжь, студенты |
| Монетизация | Success-fee 2–5% + подписки + консультант-маркетплейс | Freemium → платный тариф, upsell на consulting |
| Отличие от прошлого сезона | Реальный success-fee ledger, тесты, security-аудит | Фича не существовала год назад |

---

## Стадия 0 — до подачи заявки (человеческие задачи, не код)

- [ ] Подтвердить точный дедлайн регистрации 2026 напрямую на awards.gov.uz (сайт отдавал 503
  боту — открыть вручную в браузере) или у IT Park.
- [ ] Уточнить точный список 10 секторов трека Best Startup Project (на случай, если Incubation
  переполнен и организаторы предложат другой трек).
- [ ] Зарегистрировать команду на обе программы отдельно (участие только для граждан Узбекистана).
- [ ] Состав команды и роли для питча — кто говорит на Demo Day и Grand Final.

## Стадия 1 — Заявка (онлайн-форма)

Единый смысловой стержень для обеих форм: **проблема → почему AI → почему сейчас → трекшн →
бизнес-модель**. Не копировать текст между заявками — сегмент и метрики разные.

**Business/PTA заявка** — акцент на: реальные PSP-интеграции (не мок), success-fee как
низкорисковая для клиента модель, консультант-маркетплейс как канал расширения.

**Education/President AI Award заявка** — акцент на: каракалпакская локализация (инклюзия),
AI-guided journey (не просто каталог), сертификат как измеримый результат для молодёжи.

## Стадия 2 — Менторские сессии

Готовить заранее: 1) список из 3 конкретных вопросов по продукту, на которые хотим обратную
связь (не просить общую оценку); 2) черновик unit-экономики (CAC/LTV — хотя бы оценочно) для
Business-трека; 3) метрики завершения онбординга (completion rate по 10 шагам) для Education-трека.

## Стадия 3 — Demo Day (3 минуты, живой продукт, не слайды)

**Business-сценарий:** предприниматель заходит → AI подбирает 3 релевантных гранта/тендера за
секунды → eligibility-check с баллом → AI-черновик заявки → статус в трекере → (если won)
success-fee появляется в `/admin/success-fees` как реальная запись, не выдумка.

**Education-сценарий:** новый участник Lab → чеклист 10 шагов → карточка «следующее действие» →
каракалпакская подсказка → прогресс сертификата → вид ментора в `/admin/lab` с авто-подсказкой,
что делать дальше.

Правило: показывать **реальный UI**, не мокапы. Если что-то не готово к демонстрации — не
включать в сценарий, а не «подрисовывать».

## Стадия 4 — Fact-Checking

Жюри проверяет работоспособность и качество кода. Действие: дать доступ к репозиторию заранее
(если формат допускает) + передать [`JUDGE_EVIDENCE_PACKET.md`](./JUDGE_EVIDENCE_PACKET.md) как
единую точку входа — 132 теста, зелёный CI, 0 открытых security findings, честный список того,
что не готово (раздел 7 пакета). Не импровизировать ответы под давлением — все цифры уже
задокументированы и воспроизводимы командой `npm run typecheck && npm test && npm run test:convex`.

## Стадия 5 — Pitch Polish

Структура финального питча (обе программы, разный контент, одна структура):

1. **Проблема** (10 сек) — конкретный, узнаваемый сценарий провала без продукта.
2. **Почему сейчас** (10 сек) — President Tech/AI Award контекст, рост экосистемы.
3. **Продукт вживую** (60 сек) — сценарий из Стадии 3.
4. **Бизнес-модель** (30 сек) — конкретные цифры, не «мы верим в потенциал».
5. **Трекшн** (30 сек) — реальные метрики на момент финала (см. Стадию 0 плейбука
  «pilot-набор» — раздел ниже).
6. **Команда** (15 сек) — кто и почему именно эта команда выполнит.
7. **Ask** (5 сек) — что конкретно нужно от жюри/инвестиции.

## Стадия 6 — Grand Final

Международное жюри — питч на английском должен быть заготовлен параллельно с русским с самого
начала Pitch Polish, не переводиться в последнюю ночь.

## Трекшн-трек (человеческая задача, вне кода)

Продукт готов к реальному пилоту (payments-инфраструктура собрана и покрыта sandbox-тестами,
go-live чек-листы — [`UZUM-GO-LIVE.md`](./UZUM-GO-LIVE.md), [`PAYME-GO-LIVE.md`](./PAYME-GO-LIVE.md),
[`CLICK-GO-LIVE.md`](./CLICK-GO-LIVE.md)). Решение о глубине пилота (soft-launch без оплаты vs
реальные платящие пользователи) принимается по факту оставшегося времени до дедлайна (Стадия 0) —
это единственная переменная, которую не может закрыть код.
