# FundingPro Opportunities Lab — Onboarding Layer (Master Prompt & Acceptance Criteria)

Исполняемая спецификация («мастер-промпт») для добавления направляемого customer journey
для участников FundingPro Opportunities Lab. Слой добавляется **поверх** существующей
платформы: без переписывания auth, дашборда, роутов и без деструктивных миграций.

## Инварианты (нарушение = провал задачи)

1. Ни один существующий файл не удаляется; существующие роуты не переименовываются.
2. Изменения БД только аддитивные: одна новая Convex-таблица `labParticipants`,
   все поля опциональные, существующие таблицы не изменяются.
3. Существующий онбординг НКО (`convex/onboarding.ts`, `OnboardingChecklist`) не трогается.
4. Переиспользуются существующие данные: сохранённые гранты → «10 возможностей»,
   `applications` со статусом SUBMITTED → «заявка подана», `documents` c docType=CV → «CV загружено»,
   `proposalProjects` → черновик мотивационного письма.
5. Стиль — существующая дизайн-система (funding-green, rounded-2xl, паттерны дашборда/админки).
6. Все изменения проходят гейты репозитория: `.collect()`-аудит, `v.any()`-аудит, semgrep,
   typecheck, lint --max-warnings=0.

## Функциональный объём (MVP)

### Студент (`/dashboard/lab`)
- Чеклист из 10 шагов: регистрация → профиль → интересы → CV → LinkedIn →
  10 возможностей → мотивационное письмо → выбор 1 возможности → подача заявки → пруф.
- Состояния шага: `not_started | in_progress | submitted | needs_revision | completed`.
- Прогресс-бар с процентом (шаг засчитан при `submitted|completed`).
- Карточка «следующее действие» с кнопкой.
- Карточка «Прогресс сертификата»: профиль, CV, мотивационное письмо, LinkedIn,
  10 возможностей, 1 реальная заявка, посещаемость (ручная отметка ментора).
- Подсказки на каракалпакском под каждым шагом.
- Форма профиля Lab: ФИО, возраст, город/район, Telegram, статус обучения,
  интересы (гранты/стипендии/форумы/конкурсы/стажировки/волонтёрство/обмены/хакатоны/стартапы),
  статус CV (загружено / нужна помощь), ссылка LinkedIn, отметки о письме и пруфе.
- Сводная карточка на главной дашборда со ссылкой на `/dashboard/lab`.

### Ментор/админ (`/admin/lab`)
- Таблица участников: ФИО, Telegram, статус, по-шаговые чекбоксы
  (профиль, интересы, CV, LinkedIn, 10 возможностей, письмо, заявка), сертификат,
  заметки, следующее действие (авто-подсказка ментору).
- Статусы участника: New applicant, Registered, Onboarding incomplete, Active participant,
  Needs reminder, Strong participant, Application submitted, Completed, Dropped.
- Ручная отметка посещаемости (`attendanceOk`) — placeholder вместо полного модуля.
- Редактирование статуса/заметок/ревизий с записью в audit log.

### API (REST, в стиле /api/v1)
- `GET  /api/v1/lab/journey` — журнал шагов текущего пользователя (withActiveUser).
- `PATCH /api/v1/lab/profile` — upsert Lab-профиля (withActiveUser).
- `GET  /api/v1/admin/lab/participants` — таблица журнала (withAdmin).
- `PATCH /api/v1/admin/lab/participants/:id` — статус/заметки/посещаемость/ревизии (withAdmin, audit log).
